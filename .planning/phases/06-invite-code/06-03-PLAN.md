## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| POST /api/auth/login-password | 公开端点——接受用户名+密码，所有验证在service层 |
| POST /api/auth/register | 公开端点——接受用户名+密码+可选邀请码，用户名唯一性和密码强度在service层验证 |
| POST /api/auth/migrate | 需认证端点——通过AuthMiddleware获取当前用户ID |
| POST /api/auth/change-password | 需认证端点——旧密码在service层验证 |
| PUT /api/auth/invite-code | 需认证端点——邀请码唯一性由DB约束保证 |
| PUT /api/admin/users/:id/password | AdminMiddleware保护——管理员免旧密码 |
| GET/PUT /api/admin/invite-config | AdminMiddleware保护——config.json读写 |
