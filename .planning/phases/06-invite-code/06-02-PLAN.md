<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LoginWithPassword | 不区分用户名不存在和密码错误——统一返回"用户名或密码错误"防止用户名枚举 |
| RegisterUser | 邀请码验证 + 配额发放在同一请求流程中；用户名验证在服务层 |
| SetInviteCode | 数据库uniqueIndex确保原子性——不预先SELECT检查 |
| ChangePassword | 需要旧密码验证（用户路径），AdminResetPassword无需旧密码（管理员路径） |
| bcrypt.CompareHashAndPassword | 时序安全比较（由bcrypt库保证） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Information Disclosure | LoginWithPassword | mitigate | 用户名不存在和密码错误返回完全相同的错误消息"用户名或密码错误" |
| T-06-02-02 | Elevation of Privilege | LoginWithPassword | mitigate | bcrypt.DefaultCost (10) 每次验证约250ms，天然限速暴力破解（D-07无lockout的补偿措施） |
| T-06-02-03 | Spoofing | SetInviteCode | mitigate | 数据库级别uniqueIndex + 直接UPDATE（非check-then-set），捕获UNIQUE constraint violation返回"该邀请码已被使用" |
| T-06-02-04 | Tampering | RegisterUser配额发放 | mitigate | 使用gorm.Expr("quota + ?")原子更新，不在应用层读-改-写 |
| T-06-02-05 | Information Disclosure | 密码哈希 | mitigate | 所有密码通过bcrypt.GenerateFromPassword哈希后存储；service/models.go中PasswordHash json:"-"永不序列化 |
| T-06-02-06 | Elevation of Privilege | ChangePassword | mitigate | 用户路径必须验证旧密码；管理员路径通过AdminMiddleware授权后方可免旧密码 |
| T-06-02-07 | Repudiation | 配额变动 | accept | 当前系统无审计日志表，配额变动仅在数据库中有最终值。低风险 |
</threat_model>
