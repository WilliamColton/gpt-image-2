## 记账口径

### 计费单位
| Option | Description | Selected |
|--------|-------------|----------|
| 按实际输出图 | 只按最终保存成功的 outputImages 数量记账；和当前 used_count 递增逻辑一致，失败不收费。 | ✓ |
| 按请求 n | 按用户请求的 params.n 记账；即使部分图片保存失败也按请求数计算，逻辑简单但可能高估。 | |
| 按 API 返回图 | 按 OpenAI/API 实际返回图片数记账；比请求 n 准确，但还要处理返回成功但本地保存失败的差异。 | |
| 你决定 | 由 Claude 根据现有生成成功路径选择最一致的口径。 | |

**User's choice:** 按实际输出图
**Notes:** 成本和收入只按最终保存成功的图片张数计算。

### 部分成功
| Option | Description | Selected |
|--------|-------------|----------|
| 只记成功张数 | 成功几张就按几张记账；与“按实际输出图”完全一致，图表反映真实可交付结果。 | ✓ |
| 任务标记异常 | 记录成功张数，同时给统计记录标记 partial；后续图表/排障更细，但实现更重。 | |
| 整单不记账 | 只要不是完整成功就不记账；对用户友好但低估实际 API 成本。 | |
| 你决定 | 由 Claude 按现有任务状态和实现复杂度选择。 | |

**User's choice:** 只记成功张数
**Notes:** 部分成功不阻断统计，按已保存图片数入账。

### Failover 成本归属
| Option | Description | Selected |
|--------|-------------|----------|
| 成功端点 | 只把成本归到最终成功生成图片的端点；失败端点不计成本。 | ✓ |
| 所有尝试端点 | 每个尝试过的端点都可能产生费用；更保守但需要知道失败请求是否被供应商计费。 | |
| 手动调整 | 系统只记成功端点，失败端点成本后续人工修正；实现简单但账面不完全自动。 | |
| 你决定 | 由 Claude 根据现有 failover 逻辑和你的目标选择。 | |

**User's choice:** 成功端点
**Notes:** 失败端点不计成本。

## 定价配置

### 成本价历史
| Option | Description | Selected |
|--------|-------------|----------|
| 保存价格快照 | 端点配置里有当前成本价，但每条统计记录保存当时成本/售价；以后改价不会篡改历史利润。 | ✓ |
| 只存当前价 | 只在 apiEndpoints 上存 costPerImage，图表查询时用当前价计算；实现简单但历史数据会随改价变化。 | |
| 按日期价格表 | 维护价格生效日期和历史价格表；最准确但超出当前小后台需求复杂度。 | |
| 你决定 | 由 Claude 根据可维护性和现有 config.json 模式选择。 | |

**User's choice:** 保存价格快照
**Notes:** 当前配置用于未来任务，历史记录保存当时价格。

### 全局售价
| Option | Description | Selected |
|--------|-------------|----------|
| 系统配置+快照 | 在管理后台系统配置里设置当前售价，每条成功统计保存当时售价；改价不影响历史收入。 | ✓ |
| 只存系统配置 | 只保存当前售价，图表实时用当前售价计算历史收入；实现轻但历史收入会变化。 | |
| 按用户售价 | 未来可给不同用户不同售价；这是新能力，当前 phase 会明显扩大范围。 | |
| 你决定 | 由 Claude 按本阶段“全局售价”的范围选择。 | |

**User's choice:** 系统配置+快照
**Notes:** 售价也是全局当前值 + 每条账单历史快照。

### 金额精度
| Option | Description | Selected |
|--------|-------------|----------|
| 最多4位小数 | 单位元/张，支持如 0.0123；数据库用整数存“万分之一元”或等价精度，适合图片 API 成本。 | ✓ |
| 最多2位小数 | 按人民币分计；界面简单，但小额 API 成本可能不够精确。 | |
| 最多6位小数 | 更精确，适合极低单价；UI 和格式化略复杂。 | |
| 你决定 | 由 Claude 结合成本价常见精度选择。 | |

**User's choice:** 最多4位小数
**Notes:** 实现应避免浮点金额误差。
