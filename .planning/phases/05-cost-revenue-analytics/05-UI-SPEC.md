## Phase-Specific Visual and Interaction Contract

### Admin shell
- 在现有 `/admin` 管理后台内新增一个顶层标签页 `成本收益统计`，放在 `系统配置` 之后、`公告管理` 之前。
- 保持现有管理后台壳层不变：同一 `max-w-7xl` 宽度、同一页头、同一主题切换、同一 logout / refresh 交互。
- 不新增独立路由，不做全屏仪表盘。

### Analytics layout
- 默认时间范围为 `7天`。
- 首行展示 4 个 KPI 卡片：`总收入`、`总成本`、`利润`、`成功图片数`。桌面端 4 列、平板 2x2、移动端单列。
- 中间区域展示趋势卡片，卡片头部放时间筛选 `今日 / 7天 / 30天 / 全部` 和 `刷新统计` 按钮。
- 仅提供时间范围筛选；第一版不提供端点筛选、用户筛选、搜索、导出或钻取。
- 底部展示两个拆分卡片：`端点拆分` 与 `用户拆分`。桌面端并排，移动端上下堆叠。
- 页面只展示汇总、趋势和表格；不提供 CSV/PDF 导出，不提供点击钻取，不提供行内编辑。

### Charts
- 趋势区必须可视化 `收入 / 成本 / 利润 / 成功图片数`，但图表实现方式由 planner 决定。
- 视觉上优先采用“两层趋势”结构：上层金额趋势图（收入、成本、利润三条序列），下层成功图片数趋势图（柱状或折线均可，保持同一时间轴）。
- 图例始终可见，hover 时显示精确日期和值。
- 网格线、坐标轴和辅助线保持低对比度；只有数据序列使用彩色。
- 当所选时间范围没有数据时，不移除卡片；保留空图框和空状态文案。

### Tables
- 拆分表默认按 `利润` 降序排列，次级排序为 `收入`。
- 金额列右对齐，数字使用 tabular numbers / 等宽数字，保留最多 4 位小数。
- 表头在卡片滚动时保持可读；表格内容过长时使用卡片内滚动，不分页。
- 行只读，不能点击进入详情，不能在表格内执行删除、导出或其他操作。
- 端点表列建议为：端点标识、成功图片数、收入、成本、利润、利润率。
- 用户表列建议为：用户标识、成功图片数、收入、成本、利润、利润率。
- 负利润必须用红色数值显示；零利润用中性色；正利润用绿色数值显示。

### Pricing configuration
- 在 `系统配置` 中，为每个 API 端点新增 `成本价（元/张）` 输入框，和现有 `Base URL / API Key / 并发 / 优先级` 同级展示。
- 新增一个独立的 `全局售价（元/张）` 卡片，放在端点池卡片下方。
- 价格输入全部使用非负数字，最多 4 位小数，`step=0.0001`，默认右对齐，旁边给出 `支持 4 位小数` 的提示。
- 保存按钮统一使用单一 CTA `保存价格配置`，一次性提交所有价格字段，避免逐行保存造成历史快照不一致。
- 保存成功后刷新回显的配置值，并保留当前滚动位置。
- 价格字段为空、负数或超过 4 位小数时，保存按钮保持禁用并在字段下方显示内联错误。

### States
- Loading：首屏和切换时间范围时使用骨架屏，分别覆盖 KPI、趋势图和表格区域。
- Empty：使用 `EmptyState` 居中展示，空态文案指向“已完成图片生成后再回来查看统计”。
- Error：卡片内展示单条危险提示条，并提供 `刷新统计` / `重试保存` 的单一操作入口。
- Success：价格保存成功时使用 toast `价格配置已保存`；统计刷新成功时使用 toast `统计已刷新`。
- Partial failure：如果某一块数据拉取失败，保留其他已加载内容，不清空整页。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline adornments, fine-grain label spacing |
| sm | 8px | Compact control clusters, chip gaps |
| md | 16px | Default form groups, table cell padding, card content spacing |
| lg | 24px | Section padding inside cards, chart-to-table separation |
| xl | 32px | Gaps between KPI row and detail rows |
| 2xl | 48px | Separation between major dashboard regions |
| 3xl | 64px | Page-level breathing room and top/bottom shell spacing |

Exceptions: touch-only icon buttons and chart toolbar buttons must keep a minimum 44px hit target on mobile; visual spacing still snaps to the 8-point grid.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f4f4f5` light / `#09090b` dark | Page canvas, outer shell, chart backdrop, whitespace |
| Secondary (30%) | `#ffffff` light / `#18181b` dark with 90–95% opacity | Cards, sticky nav, tables, form surfaces |
| Accent (10%) | `#2563eb` | Active admin tab, selected time-range chip, primary save CTA, focus rings, selected row emphasis, chart highlight |
| Destructive | `#ef4444` | Delete / disable confirmations only |

Accent reserved for: active admin tab, selected time-range chip, primary save CTA, focus rings, selected row highlight, current chart series emphasis.

Chart semantic palette:
- 收入：blue-600
- 成本：amber-500
- 利润：emerald-600（负值改用 red-500）
- 成功图片数：violet-500 或 zinc-500，二选一但全页保持一致

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 保存价格配置 |
| Empty state heading | 暂无成本收益数据 |
| Empty state body | 完成图片生成并保存价格配置后，这里将显示收入、成本、利润和成功图片数。 |
| Error state | 统计数据加载失败，请点击“刷新统计”重试；保存价格失败时，请检查金额是否为数字且最多 4 位小数。 |
| Destructive confirmation | 删除端点配置：确定删除该 API 端点吗？历史账单会保留，但该端点将不再参与后续成本统计。 |
