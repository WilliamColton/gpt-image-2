**1. Create backend-go/service/log.go (NEW file)**

Package: service

Imports needed: "gpt-image-playground/backend/database", "gpt-image-playground/backend/util", "log/slog", "time"

Define these exported types and functions:

```go
// LogEventParams is the input for LogEvent
type LogEventParams struct {
    EventType   string
    UserID      string
    UserLabel   string
    Message     string
    Severity    string // "INFO", "WARN", "ERROR" — defaults to "INFO" if empty
    IP          string
    DetailsJSON string // pre-serialized JSON string
}

// LogQuery holds optional filter parameters for QueryLogs
type LogQuery struct {
    EventType string
    Severity  string
    User      string // case-insensitive LIKE match on user_label
    Keyword   string // LIKE match on message
    Range     string // "today", "7d", "30d", "all" — parsed via ParseAnalyticsRange
}
```

LogEvent(p LogEventParams):
- Default Severity to "INFO" if empty
- Create &database.AuditLog{} with: ID=util.GenerateID(), EventType=p.EventType, UserID=p.UserID, UserLabel=p.UserLabel, Message=p.Message, Severity=p.Severity, IP=p.IP, DetailsJSON=p.DetailsJSON, CreatedAt=time.Now().UnixMilli()
- Call database.DB.Create(entry); on error, slog.Error("审计日志写入失败", "event_type", p.EventType, "error", err) and return (fire-and-forget, per D-12)
- No return value. Audit logging is best-effort per RESEARCH.md open question #1.

QueryLogs(q LogQuery) ([]database.AuditLog, error):
- Start with database.DB.Model(&database.AuditLog{})
- Conditionally add WHERE clauses using GORM parameterized queries (use ? placeholder, never string concatenation):
  - q.EventType != "" → db.Where("event_type = ?", q.EventType)
  - q.Severity != "" → db.Where("severity = ?", q.Severity)
  - q.User != "" → db.Where("user_label LIKE ?", "%"+q.User+"%")
  - q.Keyword != "" → db.Where("message LIKE ?", "%"+q.Keyword+"%")
  - q.Range != "" && q.Range != "all" → call ParseAnalyticsRange(q.Range, time.Now()), on success add db.Where("created_at BETWEEN ? AND ?", r.From, r.To)
- Order by created_at DESC (per D-09: reverse chronological)
- Find into []database.AuditLog slice, return it with any error

CleanLogs(before *int64) (int64, error):
- Start with database.DB.Model(&database.AuditLog{})
- If before != nil → db.Where("created_at < ?", *before)
- Call db.Delete(&database.AuditLog{})
- Return result.RowsAffected, result.Error

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (AdminMiddleware) | Admin auth token required for GET/DELETE /api/admin/logs |
| handler→service | Query params from untrusted HTTP request flow into service.QueryLogs WHERE clause builder |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | service.QueryLogs() LIKE clauses | mitigate | GORM Where("field LIKE ?", "%"+val+"%") uses parameterized queries — ? placeholder prevents SQL injection. Verified by codebase pattern (GetBillingSummary uses same approach) |
| T-09-02 | Information Disclosure | GET /api/admin/logs | mitigate | Route is in adminAuth group protected by AdminMiddleware (token verification + admin role check). Unauthorized access returns 403 |
| T-09-03 | Spoofing | LogEvent() caller fields | accept | All primary log fields (user_id, event_type, severity, message) are server-composed from JWT identity and handler context. User input only appears in details_json values, not in primary identifying fields. Risk is low |
| T-09-04 | Tampering | DELETE /api/admin/logs | accept | D-05 intentionally allows admins to clear logs. ConfirmDialog on frontend prevents accidental clicks. Deletion itself should produce a log entry (meta-audit), but this is deferred to a future phase |
| T-09-05 | Information Disclosure | details_json field | mitigate | Service/handler code must never write passwords, API keys, or full auth tokens into DetailsJSON. Only operational metadata (task IDs, quota deltas, endpoint counts, usernames). Enforced by code review at Plan 02 instrumentation stage |
| T-09-SC | Tampering | Package installs (npm/pip/cargo) | accept | No new packages installed in this phase. All dependencies (gorm v1.30.0, gin v1.10.1) are pre-existing and verified in go.mod |
</threat_model>
