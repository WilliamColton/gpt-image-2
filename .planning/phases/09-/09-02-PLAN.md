## Trust Boundaries

|----------|-------------|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Information Disclosure | details_json content | mitigate | DetailsJSON in this plan carries only operational metadata: task IDs, quota deltas, endpoint counts, usernames, error messages. No passwords, API keys, or auth tokens. Enforced by code review |
| T-09-07 | Spoofing | Admin actor identity | mitigate | AdminMiddleware now sets c.Set("user") with the actual JWT subject ("admin") and label "admin". Admin audit entries are attributable to the correct actor |
| T-09-08 | Denial of Service | LogEvent() inline writes | accept | D-12 mandates synchronous INSERT. SQLite single-writer is the bottleneck, but quota system limits throughput. Gigantic details_json is not possible — values are bounded (task IDs are 21 chars, prompts truncated to 100 chars, error messages truncated to 200 chars) |
| T-09-09 | Elevation of Privilege | AdminMiddleware user context | accept | Adding c.Set("user") in AdminMiddleware doesn't change auth logic — it only provides context for logging. The role check (must be "admin") still runs before c.Set |
